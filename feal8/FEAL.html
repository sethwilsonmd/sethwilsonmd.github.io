FEAL Algorithm

The Fast Data Encipherment Algorithm (FEAL) was proposed by Akihiro Shimizu and Shoji Miyaguchi in 1988 as an alternative to the DES algorithm.[1] Their purpose was to create a fast symmetric encryption algorithm that did not use a lookup table and could be implemented in both hardware and software. FEAL is a block cipher that uses a 64-bit secret key (k) to create a 256-bit extended key made up of 16 16-bit subkeys (K0 to K15). This extended key is then used to transform a 64-bit plaintext message (m) into a 64-bit ciphertext message (c). The algorithm as originally presented featured four rounds of a "data randomization" function. When cryptanalysis revealed weaknesses in that design, an additional four rounds of data randomization were added. The eight round version (FEAL Version 2.00, FEAL-8) is implemented here.

Speed Test

In their paper, Akihiro Shimizu and Shoji Miyaguchi stated: "When FEAL is implemented in assembly language on an i-8086 16-bit microprocessor with 8 MHz frequency, it is confirmed that the program size is 400 bytes and the excution time speed reaches 120 kbps."
